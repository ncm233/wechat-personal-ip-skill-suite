# WeChat Style Factory

Distill a local article-and-cover archive into a reusable style library for personal IP, WeChat article production, and lead-generation content workflows.

## Inputs

Expected inputs:

- A ZIP archive or extracted folder containing article Markdown files, cover images, `articles.csv`, and optional `index.json` / raw JSON.
- Optional output directory, author/grouping preference, number of author styles, number of cover clusters, and sample limits.

If the archive is missing, ask for the file path. If the archive is already extracted, use the extracted root directly.

## Output Shape

The factory creates:

```text
style-library/
  article-styles/
    <style-id>.md
  cover-styles/
    <style-id>.md
  cover-contact-sheets/
    <style-id>.jpg
  pipeline/
    style-index.md
    integration-notes.md
```

Each article style file follows the user's style-distillation prompt:

```text
## 1. 总体风格定位
## 2. 标题风格
## 3. 开头方式
## 4. 正文结构
## 5. 语言特点
## 6. 判断方式
## 7. 结尾方式
## 8. 可复用写作规则
```

Each cover style file follows:

```text
1. 风格名
2. 具体的风格特征拆解（不包含内容本身）

English Version
1. Style Name
2. Breakdown of Style Characteristics
```

## Workflow

### Step 1: Inspect The Archive

Run the bundled script to inspect and distill:

```powershell
python $env:USERPROFILE\.codex\skills\wechat-style-factory\scripts\distill_style_library.py `
  --input "<zip-or-extracted-root>" `
  --output-dir "<target-output-dir>"
```

For the user's YouMind archive, a typical command is:

```powershell
python $env:USERPROFILE\.codex\skills\wechat-style-factory\scripts\distill_style_library.py `
  --input "E:\dowmlaod\Downloads\2026-06-15-youmind-x-viral-articles-all.zip" `
  --output-dir "D:\Users\ZhuanZ（无密码）\Documents\New project\youmind-style-library"
```

### Step 2: Article Style Distillation

The script groups articles by `author_handle` first, then generates one article-style file per top author/group. It uses title, summary, opening, headings, paragraph rhythm, list usage, and engagement metadata.

Use the generated files as draft style skills. When higher fidelity is needed, manually or agentically refine a style file by reading 20-100 samples from that group.

### Step 3: Cover Style Distillation

The script clusters cover images by visual features:

- brightness and dark-field ratio
- saturation and colorfulness
- edge density / linework density
- contrast
- image aspect and texture signals

It creates one contact sheet per cluster plus a corresponding bilingual cover-style Markdown file. Use the contact sheet to visually verify or refine the style name.

### Step 4: Pipeline Integration

Use `pipeline/style-index.md` to choose:

- an article writing style for `wechat-rewrite`
- a cover visual style for `wechat-cover-image`
- optional future custom options to add into those skills

If the user asks to "加入最终 pipeline", update `wechat-article-pipeline` so the pipeline can accept:

```text
article_style=<style-id>
cover_style=<style-id>
```

Then map those IDs to the style files generated by this factory.

## Quality Rules

- Do not include article content verbatim in the final style files except short title/opening examples when needed.
- Do not analyze article topics as style.
- Do not treat publishing boilerplate, recommended reading, repost notices, or footer disclaimers as writing style.
- For covers, analyze visual style only, not the article topic shown by the filename or title.
- Keep each style file independently reusable.
- Prefer stable, lowercase, hyphenated style IDs.

## Default Prompt

Use `wechat-style-factory` to extract article-writing styles and cover-image styles from my local archive. Generate one Markdown file per style, create cover contact sheets, and produce a style index that can be connected to my WeChat article pipeline.
